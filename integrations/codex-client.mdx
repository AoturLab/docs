本页介绍如何使用 CC Switch 为 Codex 客户端添加 HiLinkup 供应商。完成后，你可以直接在 Codex 客户端的模型菜单中选择 HiLinkup 平台提供的中国模型。

<Card title="下载 CC Switch" icon="download" href="https://ccswitch.io/">
  前往 CC Switch 官网下载与你的操作系统对应的版本。
</Card>

## 配置前准备

你需要：

- 已安装 CC Switch 和 Codex 客户端。
- 从 [HiLinkup 控制台](https://hilinkup.com/console/token) 获取的 API Key。
- HiLinkup API 请求地址：`https://hilinkup.com/v1`

<Warning>
  API Key 只应粘贴到 CC Switch 的密钥字段中。不要把真实密钥写入截图、聊天消息或项目文件。
</Warning>

## 使用 CC Switch 快速配置

<Steps>
  <Step title="切换到 Codex / ChatGPT">
    打开 CC Switch，在顶部应用切换栏中点击 OpenAI 图标，进入 Codex / ChatGPT 的供应商配置。

    <Frame caption="在 CC Switch 顶部选择 Codex / ChatGPT">
      <img src="/images/codex-client-cc-switch/01-select-codex.png" alt="CC Switch 顶部应用切换栏中的 Codex 和 ChatGPT 选项" />
    </Frame>
  </Step>

  <Step title="添加供应商">
    点击右上角橙色的 `+` 按钮，打开“添加新供应商”页面。

    <Frame caption="点击右上角的添加供应商按钮">
      <img src="/images/codex-client-cc-switch/02-add-provider.png" alt="CC Switch Codex 供应商列表右上角的添加按钮" />
    </Frame>
  </Step>

  <Step title="填写 HiLinkup 连接信息">
    按下表填写：

    | 字段 | 填写内容 |
    | --- | --- |
    | Provider Name / 服务商名称 | `HiLinkup`，也可以使用便于识别的自定义名称 |
    | Website URL / 官网地址 | `https://hilinkup.com`，可选 |
    | API Key / 密钥 | 从 HiLinkup 控制台获取的 API Key |
    | API Request URL / 请求地址 | `https://hilinkup.com/v1` |
    | Upstream Format / 上游格式 | `Responses (native)` |

    保持 **Full URL** 开关关闭。HiLinkup 原生支持 Responses API，因此上游格式选择 `Responses (native)`。

    <Frame caption="填写 HiLinkup API Key、请求地址并选择 Responses（native）">
      <img src="/images/codex-client-cc-switch/03-provider-settings.png" alt="CC Switch 中 HiLinkup 供应商名称、API Key、请求地址和 Responses native 配置" />
    </Frame>
  </Step>

  <Step title="获取可用模型">
    展开 **Advanced Options**，在 **Model Mapping** 区域点击 **Fetch Models**。CC Switch 会使用当前 API Key 从 HiLinkup 获取可用模型。

    <Frame caption="点击 Fetch Models 获取 HiLinkup 模型列表">
      <img src="/images/codex-client-cc-switch/04-fetch-models.png" alt="CC Switch 高级选项中的 Fetch Models 按钮" />
    </Frame>

    <Info>
      如果获取失败，请先检查 API Key 和 `https://hilinkup.com/v1` 是否填写正确。
    </Info>
  </Step>

  <Step title="添加模型映射">
    点击 **Add Model**，新增一行模型配置。

    <Frame caption="点击 Add Model 添加一个模型">
      <img src="/images/codex-client-cc-switch/05-add-model.png" alt="CC Switch 模型映射区域中的 Add Model 按钮" />
    </Frame>

    在 **Actual Request Model** 右侧打开下拉菜单，选择要使用的 HiLinkup 模型。**Menu Display Name** 是 Codex 客户端中显示的名称，通常可与实际请求模型保持一致。

    <Frame caption="从 HiLinkup 模型列表中选择要配置的模型">
      <img src="/images/codex-client-cc-switch/06-select-model.png" alt="CC Switch 中包含通义千问、DeepSeek 和 MiniMax 模型的下拉列表" />
    </Frame>

    你可以重复点击 **Add Model**，添加多个模型。可选模型以实际拉取结果为准，例如 `glm-5.2`、`deepseek-v4-pro` 或其他当前可用的中国模型。
  </Step>

  <Step title="保存配置">
    确认模型的显示名称和实际请求模型无误，然后点击右下角 **Save**。

    <Frame caption="保存 HiLinkup 供应商与模型配置">
      <img src="/images/codex-client-cc-switch/07-save-provider.png" alt="CC Switch 编辑供应商页面右下角的 Save 按钮" />
    </Frame>
  </Step>

  <Step title="重启 Codex 客户端并选择模型">
    完全退出并重新打开 Codex 客户端。打开输入框旁的模型菜单，选择刚才配置的 HiLinkup 模型，再发送一条简短消息测试。

    <Frame caption="在重启后的 Codex 客户端中选择 HiLinkup 模型">
      <img src="/images/codex-client-cc-switch/08-select-model-in-codex.png" alt="Codex 客户端模型菜单中显示 glm-5.2 和 deepseek-v4-pro" />
    </Frame>
  </Step>
</Steps>

<Check>
  如果模型出现在 Codex 客户端菜单中，并且可以正常返回回复，配置已经生效。
</Check>

## 常见问题

<AccordionGroup>
  <Accordion title="Codex 客户端里看不到刚添加的模型">
    先确认已经在 CC Switch 中点击 **Save**，然后完全退出并重新启动 Codex 客户端。模型列表不会在配置变更后自动刷新。
  </Accordion>

  <Accordion title="Fetch Models 获取失败">
    检查 API Key 是否有效，请求地址是否为 `https://hilinkup.com/v1`，以及 **Full URL** 是否保持关闭。也可以先参考[获取模型列表](/api-reference/models)验证密钥。
  </Accordion>

  <Accordion title="模型能显示，但发送消息失败">
    确认 **Upstream Format** 使用 `Responses (native)`，并检查所选模型是否支持 Responses API。你可以先使用 [Responses API](/api-reference/responses) 中的 cURL 示例验证模型。
  </Accordion>

  <Accordion title="应该填写哪个模型名称">
    使用 **Fetch Models** 返回的实际模型 ID，不要手动猜测。**Menu Display Name** 可以自定义，但 **Actual Request Model** 必须与 HiLinkup 返回的模型 ID 完全一致。
  </Accordion>
</AccordionGroup>
