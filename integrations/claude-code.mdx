Claude Code 使用 Anthropic 风格的接口。只有当你的 HiLinkup 账户与目标中国模型支持 [Claude Messages](/api-reference/claude-messages) 兼容调用时，才使用本页配置。

<Warning>
  OpenAI Chat Completions 可用，不代表 Anthropic Messages 接口一定可用。配置前请在控制台确认模型和接口类型。
</Warning>

<Note>
  本页示例模型使用 `deepseek-v3`。这是 HiLinkup 平台模型示例，不是要求使用海外 Claude 模型。实际可用模型以 `/v1/models` 和控制台为准。
</Note>

## 配置方式选择

| 方式 | 适用场景 | 文件位置 |
| --- | --- | --- |
| 用户级配置 | 推荐个人长期使用，所有项目生效 | macOS/Linux：`~/.claude/settings.json`；Windows：`%USERPROFILE%\.claude\settings.json` |
| 项目级本地配置 | 只在当前项目生效，不提交 Git | `.claude/settings.local.json` |
| 环境变量 | 临时验证，不改文件 | 当前终端或 PowerShell 窗口 |
| `apiKeyHelper` | 从密钥管理器或脚本读取 Key | `~/.claude/settings.json` |

## 方法一：用户级 `settings.json`

推荐方式。Claude Code 官方文档说明，用户级 settings 适合保存个人偏好、工具和 API 认证；Windows 下 `~/.claude` 对应 `%USERPROFILE%\.claude`。

<Tabs>
  <Tab title="macOS">
    1. 打开配置文件：

    ```bash
    mkdir -p ~/.claude
    nano ~/.claude/settings.json
    ```

    2. 粘贴以下内容：

    ```json
    {
      "$schema": "https://json.schemastore.org/claude-code-settings.json",
      "env": {
        "ANTHROPIC_AUTH_TOKEN": "YOUR_HILINKUP_API_KEY",
        "ANTHROPIC_BASE_URL": "https://hilinkup.com",
        "ANTHROPIC_MODEL": "deepseek-v3"
      }
    }
    ```

    3. 保存：按 `Ctrl + O`，回车，再按 `Ctrl + X` 退出。

    4. 启动：

    ```bash
    claude
    ```
  </Tab>
  <Tab title="Linux">
    1. 打开配置文件：

    ```bash
    mkdir -p ~/.claude
    nano ~/.claude/settings.json
    ```

    2. 粘贴以下内容：

    ```json
    {
      "$schema": "https://json.schemastore.org/claude-code-settings.json",
      "env": {
        "ANTHROPIC_AUTH_TOKEN": "YOUR_HILINKUP_API_KEY",
        "ANTHROPIC_BASE_URL": "https://hilinkup.com",
        "ANTHROPIC_MODEL": "deepseek-v3"
      }
    }
    ```

    3. 保存：按 `Ctrl + O`，回车，再按 `Ctrl + X` 退出。

    4. 启动：

    ```bash
    claude
    ```
  </Tab>
  <Tab title="Windows PowerShell">
    1. 创建配置目录：

    ```powershell
    New-Item -ItemType Directory -Force "$env:USERPROFILE\.claude"
    ```

    2. 打开配置文件：

    ```powershell
    notepad "$env:USERPROFILE\.claude\settings.json"
    ```

    3. 粘贴以下内容并保存：

    ```json
    {
      "$schema": "https://json.schemastore.org/claude-code-settings.json",
      "env": {
        "ANTHROPIC_AUTH_TOKEN": "YOUR_HILINKUP_API_KEY",
        "ANTHROPIC_BASE_URL": "https://hilinkup.com",
        "ANTHROPIC_MODEL": "deepseek-v3"
      }
    }
    ```

    4. 关闭并重新打开 PowerShell，然后启动：

    ```powershell
    claude
    ```
  </Tab>
</Tabs>

<Warning>
  这种方式会把 API Key 写入本机配置文件。不要截图、提交或同步 `settings.json`。
</Warning>

## 方法二：项目级本地配置

如果只想让当前项目使用 HiLinkup，不影响其他项目，使用 `.claude/settings.local.json`。官方说明该文件适合个人项目配置，不应提交到 Git。

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    mkdir -p .claude
    nano .claude/settings.local.json
    ```

    粘贴：

    ```json
    {
      "env": {
        "ANTHROPIC_AUTH_TOKEN": "YOUR_HILINKUP_API_KEY",
        "ANTHROPIC_BASE_URL": "https://hilinkup.com",
        "ANTHROPIC_MODEL": "deepseek-v3"
      },
      "permissions": {
        "deny": [
          "Read(./.env)",
          "Read(./.env.*)",
          "Read(./secrets/**)"
        ]
      }
    }
    ```

    保存后加入 `.gitignore`：

    ```bash
    printf '%s\n' '.claude/settings.local.json' >> .gitignore
    ```
  </Tab>
  <Tab title="Windows PowerShell">
    ```powershell
    New-Item -ItemType Directory -Force ".claude"
    notepad ".claude\settings.local.json"
    ```

    粘贴：

    ```json
    {
      "env": {
        "ANTHROPIC_AUTH_TOKEN": "YOUR_HILINKUP_API_KEY",
        "ANTHROPIC_BASE_URL": "https://hilinkup.com",
        "ANTHROPIC_MODEL": "deepseek-v3"
      },
      "permissions": {
        "deny": [
          "Read(./.env)",
          "Read(./.env.*)",
          "Read(./secrets/**)"
        ]
      }
    }
    ```

    保存后加入 `.gitignore`：

    ```powershell
    Add-Content .gitignore ".claude/settings.local.json"
    ```
  </Tab>
</Tabs>

## 方法三：临时环境变量

适合首次验证，不改任何配置文件。

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    export ANTHROPIC_AUTH_TOKEN="YOUR_HILINKUP_API_KEY"
    export ANTHROPIC_BASE_URL="https://hilinkup.com"
    export ANTHROPIC_MODEL="deepseek-v3"
    claude
    ```
  </Tab>
  <Tab title="Windows PowerShell">
    ```powershell
    $env:ANTHROPIC_AUTH_TOKEN = "YOUR_HILINKUP_API_KEY"
    $env:ANTHROPIC_BASE_URL = "https://hilinkup.com"
    $env:ANTHROPIC_MODEL = "deepseek-v3"
    claude
    ```
  </Tab>
</Tabs>

## 方法四：用 `apiKeyHelper` 读取密钥

如果团队使用 1Password、Vault、pass、Keychain 或内部密钥服务，建议用脚本输出 API Key，避免把密钥直接写入 JSON。Claude Code 官方说明 `apiKeyHelper` 会通过系统 shell 执行，并把输出作为模型请求认证值。

<Tabs>
  <Tab title="macOS / Linux">
    创建脚本：

    ```bash
    mkdir -p ~/.claude/bin
    nano ~/.claude/bin/hilinkup-key
    ```

    粘贴：

    ```bash
    #!/usr/bin/env bash
    printf '%s' "$HILINKUP_API_KEY"
    ```

    授权：

    ```bash
    chmod +x ~/.claude/bin/hilinkup-key
    ```

    编辑 `~/.claude/settings.json`：

    ```bash
    nano ~/.claude/settings.json
    ```

    粘贴：

    ```json
    {
      "$schema": "https://json.schemastore.org/claude-code-settings.json",
      "apiKeyHelper": "~/.claude/bin/hilinkup-key",
      "env": {
        "ANTHROPIC_BASE_URL": "https://hilinkup.com",
        "ANTHROPIC_MODEL": "deepseek-v3"
      }
    }
    ```

    启动前设置密钥：

    ```bash
    export HILINKUP_API_KEY="YOUR_HILINKUP_API_KEY"
    claude
    ```
  </Tab>
  <Tab title="Windows PowerShell">
    创建脚本：

    ```powershell
    New-Item -ItemType Directory -Force "$env:USERPROFILE\.claude\bin"
    notepad "$env:USERPROFILE\.claude\bin\hilinkup-key.cmd"
    ```

    粘贴并保存：

    ```bat
    @echo off
    <nul set /p=%HILINKUP_API_KEY%
    ```

    编辑 `settings.json`：

    ```powershell
    notepad "$env:USERPROFILE\.claude\settings.json"
    ```

    粘贴：

    ```json
    {
      "$schema": "https://json.schemastore.org/claude-code-settings.json",
      "apiKeyHelper": "%USERPROFILE%\\.claude\\bin\\hilinkup-key.cmd",
      "env": {
        "ANTHROPIC_BASE_URL": "https://hilinkup.com",
        "ANTHROPIC_MODEL": "deepseek-v3"
      }
    }
    ```

    启动前设置密钥：

    ```powershell
    $env:HILINKUP_API_KEY = "YOUR_HILINKUP_API_KEY"
    claude
    ```
  </Tab>
</Tabs>

## 验证

### 1. 先验证 HiLinkup Claude Messages 接口

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    curl https://hilinkup.com/v1/messages \
      -H "content-type: application/json" \
      -H "x-api-key: $HILINKUP_API_KEY" \
      -H "anthropic-version: 2023-06-01" \
      -d '{
        "model": "deepseek-v3",
        "max_tokens": 256,
        "messages": [
          {
            "role": "user",
            "content": "Reply with: HiLinkup Claude route is working."
          }
        ]
      }'
    ```
  </Tab>
  <Tab title="Windows PowerShell">
    ```powershell
    curl.exe https://hilinkup.com/v1/messages `
      -H "content-type: application/json" `
      -H "x-api-key: $env:HILINKUP_API_KEY" `
      -H "anthropic-version: 2023-06-01" `
      -d '{
        "model": "deepseek-v3",
        "max_tokens": 256,
        "messages": [
          {
            "role": "user",
            "content": "Reply with: HiLinkup Claude route is working."
          }
        ]
      }'
    ```
  </Tab>
</Tabs>

### 2. 再验证 Claude Code

```bash
claude
```

在 Claude Code 里输入：

```text
请只回复：HiLinkup Claude Code 配置成功
```

### 3. 最后验证完整 Agent 路径

按顺序测试：

1. 不带工具的普通问答。
2. 读取当前目录结构。
3. 允许读取一个非敏感文件。
4. 在临时分支或临时目录中执行一次小改动。

## 常见问题

<AccordionGroup>
  <Accordion title="Claude Code 仍然走官方 Claude 账号">
    检查 `ANTHROPIC_BASE_URL` 是否在启动 `claude` 的同一个终端中生效。只设置 Base URL 但没有设置有效网关凭证时，某些登录态仍可能影响请求路径。
  </Accordion>
  <Accordion title="返回 401 或 403">
    确认使用的是 HiLinkup API Key，并且当前模型支持 Claude Messages 兼容调用。不要把 OpenAI 官方 Key、Anthropic 官方 Key 和 HiLinkup Base URL 混用。
  </Accordion>
  <Accordion title="普通 cURL 成功，但 Claude Code 失败">
    Claude Code 的完整 Agent 请求会带更多上下文、工具定义和系统字段。先关闭复杂任务，只验证普通问答；如果普通问答成功，再逐步启用文件读取和命令执行。
  </Accordion>
  <Accordion title="配置文件不生效">
    用户级配置放在 `~/.claude/settings.json`；项目共享配置放在 `.claude/settings.json`；个人项目配置放在 `.claude/settings.local.json`。Windows 下 `~/.claude` 对应 `%USERPROFILE%\.claude`。修改模型后建议重启 Claude Code。
  </Accordion>
</AccordionGroup>
