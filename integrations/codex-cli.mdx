Codex CLI 支持通过 `~/.codex/config.toml` 配置自定义模型供应商。HiLinkup 作为 OpenAI 兼容服务时，应使用 `https://hilinkup.com/v1` 和 Responses 协议。

<Info>
  Codex CLI 配置字段会随版本变化。以下配置基于 Codex 的 `model_providers.<id>`、`base_url`、`env_key` 和 `wire_api` 配置方式。修改后建议运行 `codex doctor` 检查配置。
</Info>

## 接入前准备

<Steps>
  <Step title="确认 Codex CLI 可用">
    ```bash
    codex --version
    codex doctor
    ```
  </Step>
  <Step title="设置 API Key">
    ```bash
    export HILINKUP_API_KEY="YOUR_HILINKUP_API_KEY"
    ```
  </Step>
  <Step title="确认模型 ID">
    ```bash
    curl https://hilinkup.com/v1/models \
      -H "Authorization: Bearer $HILINKUP_API_KEY"
    ```

    从返回的 `data[].id` 中选择适合代码任务的模型。
  </Step>
</Steps>

## 方法一：临时命令行配置

适合首次验证，不改任何配置文件。

```bash
HILINKUP_API_KEY="YOUR_HILINKUP_API_KEY" \
codex \
  -c model='"gpt-5.4"' \
  -c model_provider='"hilinkup"' \
  -c model_providers.hilinkup.name='"HiLinkup"' \
  -c model_providers.hilinkup.base_url='"https://hilinkup.com/v1"' \
  -c model_providers.hilinkup.env_key='"HILINKUP_API_KEY"' \
  -c model_providers.hilinkup.wire_api='"responses"'
```

<Note>
  `-c` 的值按 TOML 解析，所以字符串值外面需要带引号。上面的写法可以直接复制到 shell。
</Note>

## 方法二：用户级 `~/.codex/config.toml`

适合个人长期使用。编辑 `~/.codex/config.toml`，加入：

```toml
model = "gpt-5.4"
model_provider = "hilinkup"

[model_providers.hilinkup]
name = "HiLinkup"
base_url = "https://hilinkup.com/v1"
env_key = "HILINKUP_API_KEY"
wire_api = "responses"
request_max_retries = 2
stream_max_retries = 5
stream_idle_timeout_ms = 300000
```

然后在 shell profile 中保存密钥：

```bash
printf '%s\n' 'export HILINKUP_API_KEY="YOUR_HILINKUP_API_KEY"' >> ~/.zshrc
source ~/.zshrc
```

启动：

```bash
codex
```

## 方法三：Profile 配置

如果你不想把默认 Codex 配置切到 HiLinkup，可以创建一个独立 profile。

创建 `~/.codex/hilinkup.config.toml`：

```toml
model = "gpt-5.4"
model_provider = "hilinkup"

[model_providers.hilinkup]
name = "HiLinkup"
base_url = "https://hilinkup.com/v1"
env_key = "HILINKUP_API_KEY"
wire_api = "responses"
```

启动时指定 profile：

```bash
export HILINKUP_API_KEY="YOUR_HILINKUP_API_KEY"
codex --profile hilinkup
```

非交互模式：

```bash
export HILINKUP_API_KEY="YOUR_HILINKUP_API_KEY"
codex exec --profile hilinkup "阅读 README，并列出项目启动步骤。"
```

## 方法四：项目级启动脚本

团队项目可以提交一个不含密钥的脚本，方便成员统一使用。

创建 `scripts/codex-hilinkup`：

```bash
#!/usr/bin/env bash
set -euo pipefail

: "${HILINKUP_API_KEY:?Set HILINKUP_API_KEY first}"

exec codex \
  -c model='"gpt-5.4"' \
  -c model_provider='"hilinkup"' \
  -c model_providers.hilinkup.name='"HiLinkup"' \
  -c model_providers.hilinkup.base_url='"https://hilinkup.com/v1"' \
  -c model_providers.hilinkup.env_key='"HILINKUP_API_KEY"' \
  -c model_providers.hilinkup.wire_api='"responses"' \
  "$@"
```

授权并运行：

```bash
chmod +x scripts/codex-hilinkup
export HILINKUP_API_KEY="YOUR_HILINKUP_API_KEY"
./scripts/codex-hilinkup "总结这个仓库的目录结构。"
```

## 常见问题

<AccordionGroup>
  <Accordion title="配置后仍然没有走 HiLinkup">
    检查 `model_provider = "hilinkup"` 是否设置在当前生效的配置层。使用 `--profile hilinkup` 时，文件名必须是 `~/.codex/hilinkup.config.toml`。
  </Accordion>
  <Accordion title="提示找不到 API Key">
    确认 `env_key = "HILINKUP_API_KEY"`，并且启动 Codex 的同一个 shell 中能执行 `echo $HILINKUP_API_KEY`。
  </Accordion>
  <Accordion title="Responses 路径不通">
    Codex 的自定义 provider 建议使用 `wire_api = "responses"`。先用 [Responses](/api-reference/responses) 页面里的 cURL 验证 `https://hilinkup.com/v1/responses`。
  </Accordion>
  <Accordion title="不要覆盖内置 openai provider">
    不要写 `[model_providers.openai]`。`openai` 是保留 ID；自定义供应商请使用 `hilinkup`、`hilinkup-prod` 这类新 ID。
  </Accordion>
</AccordionGroup>
