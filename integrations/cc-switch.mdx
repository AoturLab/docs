CC Switch 可以集中管理 Claude Code 的供应商配置。把 HiLinkup 添加为供应商后，你可以在 CC Switch 里切换到 HiLinkup，并把 Claude Code 的 Sonnet、Opus、Fable、Haiku 等角色映射到 HiLinkup 平台模型。

<Warning>
  CC Switch 面向 Claude Code 风格调用。配置前请确认你的 HiLinkup 账户和目标模型支持 [Claude Messages](/api-reference/claude-messages) 兼容接口。
</Warning>

<Note>
  本页示例使用 `deepseek-v4-pro`、`deepseek-v4-flash`、`MiniMax-M2.1` 等中国模型作为映射示例。实际可选模型以 CC Switch 的“获取模型列表”和 HiLinkup 控制台为准。
</Note>

## 配置前准备

你需要准备：

- 已安装并能打开 CC Switch。
- HiLinkup API Key。
- HiLinkup 请求地址：`https://hilinkup.com/v1`
- 至少一个支持 Claude Messages 兼容调用的模型。

先用 cURL 验证密钥和模型列表：

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    export HILINKUP_API_KEY="YOUR_HILINKUP_API_KEY"
    
    curl https://hilinkup.com/v1/models \
      -H "Authorization: Bearer $HILINKUP_API_KEY"
    ```
  </Tab>
  <Tab title="Windows PowerShell">
    ```powershell
    $env:HILINKUP_API_KEY = "YOUR_HILINKUP_API_KEY"
    
    curl.exe https://hilinkup.com/v1/models `
      -H "Authorization: Bearer $env:HILINKUP_API_KEY"
    ```
  </Tab>
</Tabs>

## 添加 HiLinkup 供应商

![CC Switch 供应商列表，右上角加号用于添加新供应商](/images/cc-switch/provider-list.png)

<Steps>
  <Step title="打开供应商管理">
    打开 CC Switch，在顶部应用切换器中选择 Claude Code。官方文档说明，应用切换器可在 Claude、Claude Desktop、Codex、Gemini、OpenCode、OpenClaw、Hermes 等应用之间切换。
  </Step>
  <Step title="点击添加">
    点击右上角的 `+`，进入“添加新供应商”页面。
  </Step>
  <Step title="选择添加方式">
    选择“应用专属供应商”。如果你的 CC Switch 版本支持“统一供应商”，也可以用统一供应商复用到多个应用；本页按 Claude Code 专属供应商说明。
  </Step>
  <Step title="填写基础信息">
    按下面填写：

    | 字段 | 填写内容 |
    | --- | --- |
    | 供应商名称 | `HiLinkup` |
    | 官网链接 | `https://hilinkup.com` |
    | API Key | 你的 HiLinkup API Key |
    | 请求地址 | `https://hilinkup.com/v1` |
  </Step>
  <Step title="确认认证字段">
    认证环境变量保持默认 `ANTHROPIC_AUTH_TOKEN`。CC Switch 会把你填写的 API Key 写入对应配置。
  </Step>
  <Step title="保存供应商">
    点击“添加”或“保存”。回到供应商列表后，确认 HiLinkup 已出现在列表中。
  </Step>
</Steps>

![在 CC Switch 中添加 HiLinkup 供应商](/images/cc-switch/add-provider.png)

<Info>
  请求地址填写兼容 Claude API 的服务端地址，不要以斜杠结尾。推荐使用 `https://hilinkup.com/v1`。
</Info>

## 获取模型列表并映射角色

添加供应商后，进入 HiLinkup 的“编辑供应商”页面。

![在 CC Switch 中获取模型列表并配置模型映射](/images/cc-switch/model-mapping.png)

<Steps>
  <Step title="获取模型列表">
    点击“获取模型列表”。CC Switch 官方文档说明，该功能会使用配置的 API Key 调用 OpenAI 兼容的 `/v1/models` 端点，并按类别显示模型下拉菜单。
  </Step>
  <Step title="映射 Sonnet">
    在 `Sonnet` 行选择一个适合日常编码和 Agent 任务的模型，例如 `deepseek-v4-pro`。
  </Step>
  <Step title="映射 Opus">
    在 `Opus` 行选择一个更强的推理或复杂任务模型，例如 `MiniMax-M2.1` 或控制台推荐的高能力模型。
  </Step>
  <Step title="映射 Fable 和 Haiku">
    给 `Fable`、`Haiku` 选择响应更快或成本更低的模型，例如 `deepseek-v4-flash`。
  </Step>
  <Step title="设置默认兜底模型">
    在“默认兜底模型”中选择一个稳定模型。未明确落到 Sonnet、Opus、Fable、Haiku 的请求会使用该模型。
  </Step>
  <Step title="保存配置">
    点击“保存”，再回到 CC Switch 供应商列表。
  </Step>
</Steps>

## 推荐映射示例

| Claude Code 角色 | 推荐用途 | HiLinkup 模型示例 |
| --- | --- | --- |
| Sonnet | 日常编码、文件编辑、常规 Agent 任务 | `deepseek-v4-pro` |
| Opus | 复杂重构、长上下文推理、架构分析 | `MiniMax-M2.1` |
| Fable | 快速问答、轻量编辑、低成本任务 | `deepseek-v4-flash` |
| Haiku | 快速分类、摘要、小任务 | `deepseek-v4-flash` |
| 默认兜底模型 | 未命中角色映射时使用 | `deepseek-v4-pro` |

<Tip>
  如果你不确定怎么选，先把 Sonnet 和默认兜底模型都映射到同一个稳定模型。确认 Claude Code 正常工作后，再分别优化 Opus、Fable 和 Haiku。
</Tip>

## 切换到 HiLinkup

<Steps>
  <Step title="回到供应商列表">
    在 CC Switch 首页找到 `HiLinkup` 供应商。
  </Step>
  <Step title="切换默认供应商">
    选中 HiLinkup。确认列表中的当前供应商显示为 `HiLinkup`，请求地址显示为 `https://hilinkup.com` 或 `https://hilinkup.com/v1`。
  </Step>
  <Step title="确认生效">
    CC Switch 官方文档说明，Claude Code 支持热重载，切换供应商后通常即时生效。如果你发现请求仍走旧供应商，再关闭并重新打开 Claude Code。
  </Step>
  <Step title="发送测试请求">
    在 Claude Code 中输入：

    ```text
    请只回复：HiLinkup via CC Switch 配置成功
    ```
  </Step>
</Steps>

## 验证顺序

按下面顺序测试，不要一开始就跑复杂 Agent 任务：

1. 普通文本问答。
2. 读取当前目录结构。
3. 读取一个非敏感文件。
4. 在临时目录或临时分支里执行一次小改动。
5. 再测试长上下文、工具调用和多文件修改。

## 常见问题

<AccordionGroup>
  <Accordion title="获取模型列表失败">
    检查 API Key 是否来自 HiLinkup，`请求地址` 是否为 `https://hilinkup.com/v1`。如果浏览器或系统代理拦截请求，也可能导致列表为空。
  </Accordion>

  <Accordion title="模型列表有模型，但 Claude Code 请求失败">
    `/v1/models` 返回模型可用，不代表该模型一定支持 Claude Messages 兼容调用。请换成明确支持 Claude Code/Claude Messages 的模型，或先用 [Claude Messages](/api-reference/claude-messages) 的 cURL 示例验证。
  </Accordion>

  <Accordion title="切换后仍然走旧供应商">
    Claude Code 通常会热重载配置。若切换后仍然走旧供应商，先确认 HiLinkup 已被设为当前供应商；仍不生效时，再关闭并重新打开 Claude Code。
  </Accordion>

  <Accordion title="应该填 https://hilinkup.com 还是 https://hilinkup.com/v1">
    在 CC Switch 的供应商请求地址中推荐填写 `https://hilinkup.com/v1`。如果当前 CC Switch 版本明确提示“不要带 /v1”，再按应用提示填写 `https://hilinkup.com`。
  </Accordion>

  <Accordion title="模型映射怎么选">
    先使用稳定模型：Sonnet 和默认兜底模型填同一个模型；Opus 填高能力模型；Fable、Haiku 填更快或成本更低的模型。上线后根据延迟、成本和成功率调整。
  </Accordion>
</AccordionGroup>
