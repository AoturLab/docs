HiLinkup API 使用 HTTPS 和 JSON，并提供多种常见模型协议的兼容端点。多数 OpenAI 兼容应用只需要替换 API Key、Base URL 和模型 ID；Claude 与 Gemini 风格应用需要使用各自协议的路径和鉴权头。

## Base URL

OpenAI 兼容端点：

```text
https://hilinkup.com/v1
```

Gemini 兼容端点使用 `v1beta` 路径，例如：

```text
https://hilinkup.com/v1beta/models/{model}:generateContent
```

## 认证

OpenAI 兼容接口：

```http
Authorization: Bearer YOUR_HILINKUP_API_KEY
Content-Type: application/json
```

Claude Messages 兼容接口：

```http
x-api-key: YOUR_HILINKUP_API_KEY
anthropic-version: 2023-06-01
Content-Type: application/json
```

Gemini 兼容接口：

```http
x-goog-api-key: YOUR_HILINKUP_API_KEY
Content-Type: application/json
```

## 主要端点

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `POST` | `/chat/completions` | 创建文本或多模态对话 |
| `POST` | `/responses` | 使用 OpenAI Responses 兼容格式创建响应 |
| `POST` | `/messages` | 使用 Claude Messages 兼容格式创建响应 |
| `POST` | `/v1beta/models/{model}:generateContent` | 使用 Gemini 兼容格式生成内容 |
| `GET` | `/models` | 获取当前密钥可用的模型 |
| `POST` | `/embeddings` | 创建向量表示 |
| `POST` | `/images/generations` | 创建图像 |

<Note>
  端点和参数是否可用取决于模型、账户和上游渠道。调用能力端点前，先确认模型列表与控制台展示。
</Note>

## 请求 ID

错误响应和响应头可能包含请求 ID。联系支持时提供请求 ID、时间、端点和模型 ID，但不要提供 API Key。

## 错误结构

```json
{
  "error": {
    "message": "Invalid token",
    "type": "new_api_error",
    "code": ""
  }
}
```
