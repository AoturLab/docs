不要只按模型名称选择。先确定任务需要的能力，再比较质量、延迟和成本。

## 选择流程

<Steps>
  <Step title="确认输入和输出类型">
    判断任务需要文本、图片、音频、向量还是工具调用。
  </Step>
  <Step title="获取当前模型">
    调用 [`GET /v1/models`](/api-reference/models)，只从当前 API Key 可访问的模型中选择。
  </Step>
  <Step title="建立小型评测集">
    使用 20–100 条真实业务样本比较准确率、格式遵循、延迟和成本。
  </Step>
  <Step title="配置回退">
    为关键请求准备能力相近的备用模型，并限制最大重试次数。
  </Step>
</Steps>

## 常见决策维度

| 维度 | 检查内容 |
| --- | --- |
| 能力 | 工具调用、视觉、结构化输出、推理、代码 |
| 上下文 | 输入历史、系统提示、工具定义和预期输出总量 |
| 延迟 | 首 Token 延迟和完整响应时间 |
| 成本 | 输入、输出和缓存 Token 的实际计费 |
| 稳定性 | 成功率、限流、上游可用性和回退效果 |

<Tip>
  线上默认模型应来自评测结果，不应只根据公开榜单决定。公开榜单通常无法覆盖你的提示词、工具和数据分布。
</Tip>
