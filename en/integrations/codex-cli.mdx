Codex CLI can use custom model providers through `~/.codex/config.toml`. For HiLinkup, use the OpenAI-compatible base URL and the Responses wire API.

<Note>
  Examples use the HiLinkup platform model `qwen3.7-plus`, a Chinese model example. Confirm available code models with `/v1/models` and the console.
</Note>

## Prepare

```bash
codex --version
codex doctor
export HILINKUP_API_KEY="YOUR_HILINKUP_API_KEY"
```

List models:

```bash
curl https://hilinkup.com/v1/models \
  -H "Authorization: Bearer $HILINKUP_API_KEY"
```

## Option 1: one-off command

```bash
HILINKUP_API_KEY="YOUR_HILINKUP_API_KEY" \
codex \
  -c model='"qwen3.7-plus"' \
  -c model_provider='"hilinkup"' \
  -c model_providers.hilinkup.name='"HiLinkup"' \
  -c model_providers.hilinkup.base_url='"https://hilinkup.com/v1"' \
  -c model_providers.hilinkup.env_key='"HILINKUP_API_KEY"' \
  -c model_providers.hilinkup.wire_api='"responses"'
```

## Option 2: user config

Edit `~/.codex/config.toml`:

```toml
model = "qwen3.7-plus"
model_provider = "hilinkup"

[model_providers.hilinkup]
name = "HiLinkup"
base_url = "https://hilinkup.com/v1"
env_key = "HILINKUP_API_KEY"
wire_api = "responses"
request_max_retries = 2
stream_max_retries = 5
stream_idle_timeout_ms = 300000
```

Save the API key in your shell profile:

```bash
printf "HiLinkup API Key: "
stty -echo
IFS= read -r HILINKUP_API_KEY
stty echo
printf "\n"
printf '\nexport HILINKUP_API_KEY="%s"\n' "$HILINKUP_API_KEY" >> ~/.zshrc
source ~/.zshrc
```

If you use bash, replace `~/.zshrc` with `~/.bashrc` or `~/.bash_profile`.

Then run:

```bash
codex
```

## Option 3: profile config

Create `~/.codex/hilinkup.config.toml`:

```toml
model = "qwen3.7-plus"
model_provider = "hilinkup"

[model_providers.hilinkup]
name = "HiLinkup"
base_url = "https://hilinkup.com/v1"
env_key = "HILINKUP_API_KEY"
wire_api = "responses"
```

Run:

```bash
export HILINKUP_API_KEY="YOUR_HILINKUP_API_KEY"
codex --profile hilinkup
```

Non-interactive mode:

```bash
codex exec --profile hilinkup "Read the README and list the setup steps."
```

## Verify

```bash
codex doctor
codex exec --profile hilinkup "Only read the current directory and summarize the project structure."
```

<Warning>Do not define `[model_providers.openai]`. `openai` is a reserved built-in provider ID. Use a custom ID such as `hilinkup`.</Warning>
