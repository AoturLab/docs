Any application that supports a custom OpenAI or OpenAI-compatible provider can generally connect to HiLinkup.

## Connection values

| Setting | Value |
| --- | --- |
| Provider type | OpenAI / OpenAI-compatible / Custom OpenAI |
| API key | A dedicated key from the HiLinkup console |
| Base URL | `https://hilinkup.com/v1` |
| Model | An exact ID returned by `GET /v1/models` |

<Warning>Unless an application explicitly requests a full endpoint, do not append `/chat/completions` to the base URL.</Warning>

## Choose the correct protocol

Not every “compatible” client uses the same protocol. Check whether the application asks for OpenAI, Anthropic/Claude, or Gemini configuration.

| Protocol | Use case | Base URL / endpoint | Auth | Reference |
| --- | --- | --- | --- | --- |
| OpenAI Chat Completions | Most chat clients, Dify, LangChain, older OpenAI SDK calls | `https://hilinkup.com/v1` | `Authorization: Bearer $HILINKUP_API_KEY` | [Chat Completions](/en/api-reference/chat-completions) |
| OpenAI Responses | New apps, agents, unified input/output flows | `https://hilinkup.com/v1` | `Authorization: Bearer $HILINKUP_API_KEY` | [Responses](/en/api-reference/responses) |
| Claude Messages | Anthropic SDKs and Claude-style clients | `https://hilinkup.com/v1/messages` | `x-api-key: $HILINKUP_API_KEY` | [Claude Messages](/en/api-reference/claude-messages) |
| Gemini generateContent | Gemini SDKs and Google AI-style clients | `https://hilinkup.com/v1beta/models/{model}:generateContent` | `x-goog-api-key: $HILINKUP_API_KEY` | [Gemini generateContent](/en/api-reference/gemini-generate-content) |

<Tip>
  If the app says OpenAI-compatible, use `https://hilinkup.com/v1`. If it says Anthropic, Claude, or Gemini, use the corresponding protocol page instead of forcing the OpenAI base URL.
</Tip>

## Integration workflow

<Steps>
  <Step title="Create a dedicated API key">
    Use a separate key for each application and environment.
  </Step>
  <Step title="List models">
    ```bash
    curl https://hilinkup.com/v1/models \
      -H "Authorization: Bearer $HILINKUP_API_KEY"
    ```
  </Step>
  <Step title="Select an OpenAI-compatible provider">
    Enter the key, base URL, and an exact model ID.
  </Step>
  <Step title="Test plain chat">
    Send a short text message before enabling images, tools, knowledge bases, or long context.
  </Step>
  <Step title="Validate advanced capabilities separately">
    Test streaming, vision, tools, and structured output only with models that support them.
  </Step>
</Steps>

## Acceptance checklist

- Connection test returns HTTP `200`
- The model is present in `/v1/models`
- Plain text chat returns a non-empty response
- Streaming completes normally
- Usage appears in the console
- Credentials are not exposed in client bundles, logs, or screenshots

## cURL templates

<Tabs>
  <Tab title="OpenAI Chat">
    ```bash
    curl https://hilinkup.com/v1/chat/completions \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer $HILINKUP_API_KEY" \
      -d '{
        "model": "gpt-4o",
        "messages": [
          {
            "role": "developer",
            "content": "You are a helpful assistant."
          },
          {
            "role": "user",
            "content": "Hello!"
          }
        ]
      }'
    ```
  </Tab>
  <Tab title="Responses">
    ```bash
    curl https://hilinkup.com/v1/responses \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer $HILINKUP_API_KEY" \
      -d '{
        "model": "gpt-5.4",
        "input": "Tell me a three sentence bedtime story about a unicorn."
      }'
    ```
  </Tab>
  <Tab title="Claude">
    ```bash
    curl https://hilinkup.com/v1/messages \
      -H "content-type: application/json" \
      -H "x-api-key: $HILINKUP_API_KEY" \
      -H "anthropic-version: 2023-06-01" \
      -d '{
        "model": "claude-sonnet-4-6",
        "max_tokens": 1024,
        "messages": [
          {
            "role": "user",
            "content": "Hello, Claude"
          }
        ]
      }'
    ```
  </Tab>
  <Tab title="Gemini">
    ```bash
    curl "https://hilinkup.com/v1beta/models/gemini-3.1-pro-preview:generateContent" \
      -H "x-goog-api-key: $HILINKUP_API_KEY" \
      -H "Content-Type: application/json" \
      -X POST \
      -d '{
        "contents": [
          {
            "parts": [
              {
                "text": "Find the race condition in this multi-threaded C++ snippet: [code here]"
              }
            ]
          }
        ]
      }'
    ```
  </Tab>
</Tabs>

## Common failures

<AccordionGroup>
  <Accordion title="cURL works but the application fails">
    Inspect the final URL. The correct chat endpoint is `/v1/chat/completions`; avoid missing or duplicated `/v1`.
  </Accordion>
  <Accordion title="The model list is empty">
    Add an exact model ID manually. Some clients cannot discover models from custom providers.
  </Accordion>
  <Accordion title="Chat works but vision or tools fail">
    Verify model capability and the application's model capability flags. Authentication is already working.
  </Accordion>
</AccordionGroup>
